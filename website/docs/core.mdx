## 安装

```bash
$ npm i sacss

# 推荐使用👇
$ yarn add sacss
```

## 目录结构

```bash
.
├── less                   # 存放 LESS Mixin 的文件夹
│   ├── SACSS.less         # 帮助创建 sacss 的辅助 LESS Mixin
│   ├── WidthPrefix.less   # 为核心代码添加自定义前缀
│   └── helpers.less       # 独立于 sacss 之外的常用辅助 Mixin
├── sass                   # 存放 LESS Mixin 的文件夹
│   ├── SACSS.scss         # 帮助创建 sacss 的辅助 SASS Mixin
│   ├── WidthPrefix.scss   # 为核心代码添加自定义前缀
│   └── helpers.scss       # 独立于 sacss 之外的常用辅助 mixin
└── index.css              # 所有项目基本相同的核心代码（只推荐定制，命名前缀）;
```

`less` 和 `sass` 拥有完全相同的文件和 API，只是语法不一样的实现而已。

核心文件的使用非常的简单，只需要**全局**引入 `sacss/index.css` 文件即可。

`sacss/index.css` 包含了，大部分构建 UI **结构** 所需要的零件，比如：

- `display` 布局
- `float` 布局
- `flex` 布局
- `position` 布局
- `font-weight` 100～900

... 更多请查看源文件。

整体核心代码未压缩只有 `2kb`, 体积非常小所以请**全局**引入。

> ️注意：只能全局使用 SACSS，不要使用 CSS module 的方式引用

## ⚠️ 命名规则

1. 只取首字母: `.db{ display:block; }`;
2. 有数字直接连接: `.mb10{ margin-bottom:10px; }`;
3. 负值直接 `-` 连接: `.mb-10{ margin-bottom:-10px; }`;
4. 特殊符号直接用 `\` 转义:  `.w100\%{ width:100%; } .lh1\.2{ line-height:1.2 }`;
5. 其它或自定义样式，用 `_`分割 `.c_fff{ color:#fff; } .bc_fff{ background-color: #ffffff; }`
6. hover 样式，用 `\:h`分割 `.fs12{ font-size:12px; } .fs12\:h:hover{ font-size:12px; }`

这套**命名规则**已经使用快接近 10 年了，是整个项目的**核心**。

一套会让人**加速**，甚至是产生**愉悦**的**命名规则**。

| `margin: 12px;`      | `.m12`   | `.M(12px)`  | `.m-4`            | `.margin-12`         | `.m2`            | `.ma3`     |
| `text-align: center` | `.tac`   | `.Ta(c)`    | `.text-center`    | `.text-align-center` | `.center`        | `.tc`      |
| `margin: -12px`      | `.m-12`  | `.M(-12px)` | `.-m-4`           | `.margin-n12`        | `.mxn2`          | `.na3`     |
| `font-size: 25px`    | `.fs25`  | `.Fz(25px)` | `.text-2xl`       | `.font-size-25`      | `.h2`            | `.f3`      |
