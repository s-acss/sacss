## Installation

```bash
$ npm i sacss

# Recommended 👇
$ yarn add sacss
```

## Tree

```bash
.
├── less                   # The folder where the LESS Mixin is stored
│   ├── SACSS.less         # Helps to create the auxiliary LESS Mixin for SACSS
│   ├── WidthPrefix.less   # Adding custom prefixes to core code
│   └── helpers.less       # Common auxiliary mixin independent of SACSS
├── sass                   # The folder where the LESS Mixin is stored
│   ├── SACSS.scss         # Helps to create the auxiliary SASS Mixin for SACSS
│   ├── WidthPrefix.scss   # Adding custom prefixes to core code
│   └── helpers.scss       # Common auxiliary mixin independent of SACSS
└── index.css              # Basically the same core code for all projects (only recommended for customization, naming prefixes);
```

`less` and `sass` have exactly the same documentation and API, just a different implementation with a different syntax.

The core file is very simple to use, just **global** introduce the `sacss/index.css` file.

`sacss/index.css` contains, for the most part, the parts needed to build the UI **structure**, such as

- `display` layout
- `float` layout
- `flex` layout
- `position` layout
- `font-weight` 100 to 900

... Please check the source file for more information.

The overall core code is only `2kb` uncompressed, very small size so please **global** introduce it.

> ️️Note: Only use SACSS globally, do not use the CSS module reference

## Naming Rules

1. with initials only: `.db{ display:block; }`;
2. with numbers directly connected: `.mb10{ margin-bottom:10px; }`;
3. with negative values directly connected with `-`: `.mb-10{ margin-bottom:-10px; }`;
4. special symbols directly escaped with `\`: `.w100\%{ width:100%; } .lh1\.2{ line-height:1.2 }`;
5. other or custom styles, split by `_` `.c_fff{ color:#fff; } .bc_fff{ background-color: #ffffff; }`
6. hover style, split by `\:h` `.fs12{ font-size:12px; } .fs12\:h:hover{ font-size:12px; }`

This set of **naming rules** has been in use for almost 10 years and is the **core** of the entire project.

A set of **naming rules** that will **accelerate** and even **pleasure**.

## Naming Comparison

| Declarations         | SACSS     | ACSS       | Tailwind CSS      | Blowdrycss           | Basscss          | Tachyons   |
| -------------------- | -------- | ----------- | ----------------- | -------------------- | ---------------- | ---------- |
| `margin: 12px;`      | `.m12`   | `.M(12px)`  | `.m-4`            | `.margin-12`         | `.m2`            | `.ma3`     |
| `text-align: center` | `.tac`   | `.Ta(c)`    | `.text-center`    | `.text-align-center` | `.center`        | `.tc`      |
