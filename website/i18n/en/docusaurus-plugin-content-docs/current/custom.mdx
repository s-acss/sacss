# Custom Token

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

The core code `sacss/index.css` is basically all **layout** related code.

There are only the following four colors, and even the very common `font-size` related code is not included.

```css
.bc_fff{background-color:#fff}
.bc_000{background-color:#000}
.c_000{color:#000}
.c_fff{color:#fff}
```

The reason for this is that front-end code from **design specifications** related to the need for **customization**.

## Design specifications

- A project design specification: all spacing, font size must be a multiple of 4.
- B project design specifications: all spacing, font size must be a multiple of 5;

Different designers have different **design specifications** for different projects.

When we create SCSS code for these **design specifications**, we can use the Mixin `SACSS` to quickly create the relevant code.

## SACSS API

<Tabs
  defaultValue="lessApi"
  values={[
    { label: 'LESS', value: 'lessApi', },
    { label: 'SASS', value: 'sassApi', }
  ]
  }
>
<TabItem value="lessApi">

```css
@import 'sacss/less/SACSS.less';

#SACSS('attribute name'; 'abbreviation name'; 'list of values'; 'units');

// attribute name: write attribute name directly `font-size` (without quotes)
// abbreviated name: abbreviated name `fs` (without quotes)
// list of values: corresponding list of values `8, 16, 24` (comma separated)
// Units: default `1px`, if needed based on rem please write the expression `1/16*1rem`
```

</TabItem>
<TabItem value="sassApi">

```css
@import 'sacss/sass/SACSS.scss';

@include SACSS('attribute name', 'abbreviation name', ('list of values'), 'units');

// attribute name: write attribute name directly `font-size` (without quotes)
// abbreviated name: abbreviated name `fs` (without quotes)
// list of values: corresponding list of values `8, 16, 24` (semicolon separated)
// Units: default `1px`, if needed based on rem please write the expression `1/16*1rem`
```
</TabItem>
</Tabs>

For how to abbreviate the naming, please [click here](./core#naming-rules).

> SACSS only works for numeric values. For example, `font-weight:bold;` cannot be created using SACSS, but `font-weight:400;` ok.


## What needs to be customized?

- `margin`: `m`
- `padding`: `p`
- `font-size`: `fs`
- `line-height`: `lh`
- `color`: `c_`
- `background-color`: `bc_`
